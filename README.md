# Task Management Backend API (SQLite)

## Description
A simple backend REST API built using Python and FastAPI that performs CRUD (Create, Read, Update, Delete) operations on tasks.  
The application uses SQLite as the database to ensure data persistence across server restarts.

This project is developed as part of an Intern / Beginner Backend Assignment.

---

## Python Version
Python 3.14.2

---
## How to Run

1. Install dependencies  
   pip install fastapi uvicorn

2. Start the server  
   uvicorn main:app --reload

3. Open API documentation  
   http://127.0.0.1:8000/docs

Swagger UI is automatically generated by FastAPI and is used to test all API endpoints.

---

## How to Work with the API

The API can be tested using the automatically generated Swagger UI.

1. Start the FastAPI server using:  
   uvicorn main:app --reload

2. Open a browser and navigate to:  
   http://127.0.0.1:8000/docs

3. Use the following endpoints to interact with the API:

   - GET /  
     Checks whether the API is running.

   - POST /tasks  
     Creates a new task.  
     Sample request body:
     ```json
     {
       "title": "Learn FastAPI",
       "description": "Backend internship assignment"
     }
     ```

   - GET /tasks  
     Retrieves all tasks stored in the database.

   - PUT /tasks/{task_id}  
     Updates the status of a task.  
     Sample request body:
     ```json
     {
       "status": "completed"
     }
     ```

   - DELETE /tasks/{task_id}  
     Deletes a task permanently from the database.

## API Endpoints

| Method | Endpoint | Description |
|------|---------|------------|
| GET | / | Health check |
| POST | /tasks | Create a new task |
| GET | /tasks | Retrieve all tasks |
| PUT | /tasks/{task_id} | Update task status |
| DELETE | /tasks/{task_id} | Delete a task |

---

## Task Data Model
Each task contains the following fields:
- id – Integer (auto-generated)
- title – String (required)
- description – String (optional)
- status – pending or completed
- created_at – Timestamp

---

## Database
- SQLite is used as the database
- Database file: tasks.db
- The database file is created automatically when the application runs
- Data persists even after restarting the server

---

## Business Rules
- Task title is mandatory
- Default task status is set to pending
- Only the status field can be updated
- Completed tasks cannot be updated
- Invalid task IDs return appropriate error responses
- Proper HTTP status codes are used

---

## Assumptions
- SQLite is used as allowed in the task description
- No authentication or authorization is required
- The database file (tasks.db) is excluded from version control
- The application is intended for learning and evaluation purposes
